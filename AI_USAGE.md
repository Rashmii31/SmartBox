# AI Usage & Development Log

This document details the usage of artificial intelligence tools during the development of the SmartBox AI system, following best practices for AI-assisted engineering and critical code review.

### 1. Which AI tool(s) you used
* **Gemini (DeepMind):** Utilized as the primary coding assistant for structural generation, Django ORM modeling, API routing, and initial layout generation due to its deep contextual integration.
* **Claude (Anthropic):** Leveraged specifically for cross-examining complex logical constraints (the 3D box packing mathematical algorithm) and refining the frontend CSS aesthetics, as Claude excels at advanced spatial reasoning and UI design.

### 2. The prompts you gave
* **Prompt 1 (Gemini):** *"Act as a Senior Django Architect. We are building an ecommerce Box Selection System. Provide the optimal database schema separating Product, Box, Order, and OrderItem. Ensure we avoid floating-point inaccuracies for warehouse metrics."*
* **Prompt 2 (Claude):** *"Review this Django Box Selection algorithm generated by Gemini. It currently only checks volumetric capacity. I need you to update the algorithm to perform a true 3D physical fit check (handling arbitrary rotations) to ensure no item dimension physically exceeds the box dimensions."*
* **Prompt 3 (Gemini):** *"Generate the DRF API endpoint and an interactive Vanilla HTML/CSS frontend. Do not use US Dollars. Localize all currency rendering to Indian Rupees (INR/₹) and use a dark-mode, premium glassmorphism design."*
* **Prompt 4 (Gemini):** *"The frontend currently recalculates the box on every keystroke. This is inefficient and bad for server load. Refactor the UI to use a dedicated 'Find Best Box' submit button."*
* **Prompt 5 (Gemini & Claude):** *"The API is returning a 'No Box Found' error even though the logic is sound. Let's debug this step-by-step. What state is the database currently in when the server boots?"*

### 3. What output you accepted
* **Database Schema:** Accepted Gemini's architecture recommendation to strictly use `DecimalField` across all models for dimensions, weights, and costs to guarantee industry-standard precision.
* **Frontend Base:** Accepted the core dark-mode CSS variables and CSS-Grid layout structure provided by Gemini.
* **3D Rotation Logic:** Accepted Claude's python loop that sorts `[length, width, height]` in descending order for both the box and the item, enabling a foolproof 3D fit comparison regardless of how the item is rotated.

### 4. What output you rejected or modified
* **Rejected Generic Currency Formatting:** The AI initially hardcoded `$` symbols and generic float logic for currency. I rejected this output and manually enforced strict INR (`₹`) formatting in the frontend and `Decimal` casting in the backend to ensure accurate localization for the Indian market.
* **Rejected Inefficient API Calls:** Gemini's first frontend draft auto-calculated the best box every time a quantity was adjusted. I modified the JavaScript to require an explicit button click, optimizing performance by preventing REST API spam.
* **Modified Database Interactions:** Claude suggested saving the user's arbitrary test items to the database *before* running the packing algorithm. I rejected this and rewrote the endpoint to calculate strictly in-memory so the frontend acts as a lightweight calculator without bloating the database.

### 5. Any mistakes the AI made
* **Database State Oversight:** The AI generated flawless Django test cases using mock database boxes, but it entirely forgot that the actual local SQLite development database was empty. This caused the live server to mysteriously throw a "No Box Found" error during manual testing. I stepped in, identified the empty inventory issue, and wrote a custom `seed.py` script to populate the local database with standard shipping boxes.
* **Quantity Multiplication Bug:** In an early draft of the algorithm, the AI forgot to multiply the item's individual weight and volume by the `quantity` field passed from the frontend (meaning an order of 10 laptops was calculated as the weight of 1 laptop). I caught this during my code review and fixed the mathematical loops.

### 6. How you verified the final code
* **Automated Testing:** I commanded the AI to generate a comprehensive Django test suite (`box_selector/tests.py`). I executed `python manage.py test` to ensure 100% pass rates across all critical scenarios: Happy Path, Weight Limit overload, Awkward Dimensions bypass, and generic Edge Cases.
* **Manual QA & Console Logging:** Spun up the local Django server and manually interacted with the DOM, adding/removing items and monitoring the browser's Network Tab to verify the API payload exactly matched the expected JSON format.
* **Cross-Verification:** I passed Gemini's python algorithms into Claude to double-check edge cases and vice-versa, utilizing multi-LLM cross-verification to ensure the spatial sorting logic was mathematically sound.
